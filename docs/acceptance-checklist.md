# 手动验收清单

更新时间：2026-03-26  
适用范围：当前 `myhome` 项目  
目标：按固定顺序验证“安装、初始化、登录、设置、内容、媒体、模板、账号维护”是否都正常工作。

## 1. 环境准备

确认以下条件成立：

1. PostgreSQL 服务可连接
2. `.env` 中 `DATABASE_URL`、`NEXTAUTH_URL`、`AUTH_SECRET` 已填写
3. 已执行：
   `npm run prisma:generate`
4. 已执行数据库迁移：
   `npx prisma migrate deploy`
5. 已执行开发初始化：
   `npm run dev:setup`

## 2. 启动验收

执行：

```bash
npm run dev
```

确认：

1. 前台首页：`http://127.0.0.1:3000/` 可访问
2. 后台登录页：`http://127.0.0.1:3000/admin/login` 可访问

## 3. 登录验收

使用默认开发账号：

1. 邮箱：`admin@example.com`
2. 密码：`Admin123456!`

确认：

1. 可以从 `/admin/login` 登录
2. 登录后进入 `/admin`
3. 未登录时访问 `/admin/*` 会被重定向到登录页

## 4. Dashboard 验收

打开 `/admin`

确认：

1. 统计卡片显示真实数量
2. 当前模板能显示
3. 最近动作有真实记录
4. 当前站点摘要能显示首页会读取的真实内容

## 5. Settings 验收

打开 `/admin/settings/site`

确认：

1. 可以修改站点名称
2. 可以修改 SEO 标题和描述
3. 可以修改页脚文案和备案信息
4. 保存后提示成功
5. 刷新页面后数据仍然存在
6. 回到首页 `/`，内容已更新

### 管理员账号验收

打开 `/admin/settings/account`

确认：

1. 可以修改显示名称
2. 可以修改登录邮箱
3. 可以修改密码
4. 修改登录邮箱或密码时会校验当前密码
5. 保存后后台会显示最新账号信息

## 6. Content 验收

打开以下子页：

1. `/admin/content/structure`
2. `/admin/content/hero`
3. `/admin/content/projects`
4. `/admin/content/skills`
5. `/admin/content/contacts`

确认以下块都可正常编辑并保存：

1. 模块开关
2. 导航菜单
3. Hero
4. 项目
5. 技能分组与技能项
6. 联系方式

### 新增 / 删除验收

确认：

1. 可以新增项目
2. 可以删除项目
3. 可以新增技能分组
4. 可以新增技能项
5. 可以删除技能项
6. 可以新增联系方式
7. 可以删除联系方式

### 排序验收

确认：

1. 项目有 `上移 / 下移`
2. 技能分组有 `上移 / 下移`
3. 技能项有 `上移 / 下移`
4. 联系方式有 `上移 / 下移`
5. 排序后对应列表顺序更新，并有即时反馈

### 媒体联动验收

确认：

1. 项目支持 `封面来源`
2. 项目支持 `本地封面`
3. 联系方式支持 `图标来源`
4. 联系方式支持 `本地图标`
5. 页面显示封面和图标预览

## 7. Media 验收

打开 `/admin/media`

确认：

1. 可以上传图片
2. 上传后列表出现新资源
3. 资源有引用统计
4. 被引用的资源不能直接删除
5. 未引用的资源可以删除

## 8. Templates 验收

打开 `/admin/templates`

确认：

1. 模板列表正常显示
2. 可以切换当前模板
3. 可以修改模板配置
4. 可以保存模板配置
5. 右侧可以看到当前站点预览
6. 模板变更保存后会反映到首页

## 9. 最终验收标准

当前版本达到通过标准时，应满足：

1. 服务可启动
2. 后台可登录
3. 核心内容可增删改排
4. 媒体可上传并被业务使用
5. 模板可切换并配置
6. 管理员账号可维护
7. 前台首页直接读取最新保存内容
