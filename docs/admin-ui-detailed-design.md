# 后台页面详细设计说明

更新时间：2026-03-25  
适用范围：个人主页系统后台管理界面  
文档目标：把后台的视觉风格、页面结构、关键交互、桌面端与手机端适配细化到可直接进入设计与开发。

## 1. 设计目标

后台不是普通管理面板，而是这套个人主页系统的一部分产品体验。  
它要解决的不是“能不能改内容”，而是：

1. 改内容时是否顺手
2. 看模板时是否有质感
3. 发布时是否有把控感
4. 在电脑端和手机端是否都保持体面和稳定

后台设计必须满足以下目标：

1. 看起来像“创作者工作台”，不是企业 OA。
2. 与前台站点有品牌关联，但不照搬前台炫光效果。
3. 桌面端重点做强，平板端可用，手机端不难用。
4. 关键页面打开后，用户能快速找到当前站点状态、内容位置和发布入口。

## 2. 总体视觉方向

### 2.1 视觉关键词

后台整体风格采用：

1. 克制
2. 高级
3. 技术感
4. 有层次
5. 不廉价

后台不做成以下类型：

1. 默认白底蓝按钮后台
2. 大量表格堆叠的工具页
3. 过度赛博霓虹的炫技页面
4. 靠厚阴影和夸张渐变硬撑高级感的界面

### 2.2 风格基调

后台沿用前台的品牌气质，但收敛成更适合长期操作的工作界面。

具体表现：

1. 使用深石墨灰和深蓝灰作为基础背景
2. 使用青蓝到紫色渐变作为品牌强调色
3. 发光效果只出现在关键操作和模板重点区域
4. 卡片与面板层级靠颜色、边框、留白和模糊，而不是大量阴影

## 3. 设计 Token 定案

以下为后台的基础设计语言，作为 UI 实现参考。

### 3.1 颜色

```text
--bg-base: #0b1020
--bg-elevated: #12182b
--bg-panel: rgba(18, 24, 43, 0.82)
--bg-soft: rgba(255, 255, 255, 0.04)
--line-subtle: rgba(255, 255, 255, 0.08)
--line-strong: rgba(255, 255, 255, 0.16)
--text-primary: #edf2ff
--text-secondary: #9ca8c7
--text-muted: #6f7c9c
--brand-cyan: #22d3ee
--brand-blue: #4f8cff
--brand-violet: #8b5cf6
--brand-gradient: linear-gradient(135deg, #22d3ee 0%, #4f8cff 45%, #8b5cf6 100%)
--success: #22c55e
--warning: #f59e0b
--danger: #ef4444
```

### 3.2 字体

后台字体组合：

1. 正文：高可读无衬线字体
2. 局部标签、数字、状态：技术感等宽字体

建议：

1. 正文：`Inter` 或同级现代无衬线
2. 等宽：`JetBrains Mono`

使用规则：

1. 页面标题、正文、表单说明用无衬线
2. 状态、模板 ID、版本号、技术标签、路径信息用等宽字体

### 3.3 圆角

```text
--radius-xs: 8px
--radius-sm: 12px
--radius-md: 16px
--radius-lg: 20px
--radius-xl: 28px
```

使用原则：

1. 小按钮和输入框使用 `8-12px`
2. 卡片和抽屉使用 `16-20px`
3. 模板展示卡和大面板可使用 `20-28px`

### 3.4 间距

```text
--space-1: 4px
--space-2: 8px
--space-3: 12px
--space-4: 16px
--space-5: 20px
--space-6: 24px
--space-8: 32px
--space-10: 40px
--space-12: 48px
```

规则：

1. 面板内部以 `16 / 20 / 24` 为主
2. 大区块之间以 `32 / 40 / 48` 为主
3. 不使用过小碎间距，避免后台显得拥挤

### 3.5 阴影与模糊

```text
--shadow-soft: 0 12px 36px rgba(0, 0, 0, 0.24)
--shadow-panel: 0 18px 60px rgba(0, 0, 0, 0.32)
--glow-brand: 0 0 24px rgba(34, 211, 238, 0.16)
--blur-panel: blur(18px)
```

规则：

1. 后台以柔和阴影为主，不做大面积强辉光
2. 毛玻璃效果仅用于导航、浮层、抽屉、模板重点区

## 4. 端侧适配策略

### 4.1 断点

```text
mobile: 360-430
tablet: 768
laptop: 1024
desktop: 1280
wide: 1440+
```

### 4.2 桌面端策略

桌面端是后台主战场，优先保证：

1. 三栏编辑效率
2. 模板预览完整度
3. 视觉空间感
4. 面板层级清晰

### 4.3 平板端策略

平板端不做桌面端强缩放，而是调整为：

1. 左侧模块树可折叠
2. 右侧预览改为抽屉或切换视图
3. 大部分页面保持双栏或单栏

### 4.4 手机端策略

手机端重点是“能完成必要管理”，不是复制桌面后台。

手机端原则：

1. 单列布局
2. 顶部固定操作栏
3. 预览与编辑切换展示
4. 复杂拖拽弱化或取消
5. 卡片点击区域足够大

## 5. 整体框架布局

### 5.1 桌面端框架

后台全局布局：

```text
左侧：品牌导航栏
顶部：全局页头
中部：页面主工作区
右侧：预览区 / 辅助面板
底部：不单独固定底栏
```

### 5.2 左侧导航栏

尺寸建议：

1. 宽度：`260px`
2. 收缩态宽度：`88px`

结构：

1. 顶部品牌区
2. 主导航分组
3. 当前模板信息
4. 底部账号区

风格：

1. 半透明深色面板
2. 轻模糊
3. 当前导航项有品牌渐变和微发光

一级导航固定为：

1. 总览
2. 内容
3. 模板
4. 媒体
5. 发布
6. 设置

### 5.3 顶部页头

结构：

1. 当前页面标题
2. 页面简述
3. 右侧快捷操作

右侧常驻操作：

1. 保存草稿
2. 预览
3. 发布

设计要求：

1. 页头始终固定在内容上方
2. 主操作按钮位置一致
3. 状态标签清晰显示 `未保存 / 草稿 / 已发布`

### 5.4 主工作区

页面主体以卡片和面板组合构成，不直接平铺表单。

规则：

1. 表单按内容分组
2. 高优先级信息在上方
3. 辅助说明与提示不占主视觉

### 5.5 右侧预览区

右侧预览是后台核心体验之一。

规则：

1. 桌面端大部分编辑页保留预览区
2. 预览区支持局部刷新
3. 预览区内页模拟前台效果，但不完全复制前台布局
4. 预览区顶部有 `桌面 / 手机` 切换

## 6. 页面详细设计

### 6.1 后台登录页

目标：简单、克制、品牌感明确。

桌面布局：

1. 左侧为品牌视觉区
2. 右侧为登录卡片

手机布局：

1. 只保留单卡片登录结构
2. 品牌视觉简化为顶部标识和一句说明

视觉元素：

1. 背景使用深色渐变与弱网格纹理
2. 中央卡片带轻微模糊和边框
3. 登录按钮使用品牌渐变

字段：

1. 邮箱
2. 密码
3. 登录按钮

不增加额外复杂项：

1. 不做社交登录
2. 不做注册入口
3. 不做找回密码复杂流程

### 6.2 Dashboard 总览

目标：一进入后台就看到当前站点状态。

桌面布局：

1. 第一行：欢迎区 + 当前模板卡
2. 第二行：内容概况卡片组
3. 第三行：最近操作 + 最近发布
4. 右侧或下方：站点缩略预览

核心模块：

1. 欢迎区
2. 当前模板
3. 发布状态
4. 内容统计
5. 快捷入口
6. 最近操作
7. 最近版本

内容统计建议显示：

1. 技能组数量
2. 项目数量
3. 联系方式数量
4. 本地媒体数量

快捷入口：

1. 编辑首屏
2. 管理项目
3. 打开模板中心
4. 进入发布中心

视觉重点：

1. 欢迎区使用大留白和品牌背景图形
2. 当前模板卡片做得像精选主题卡
3. 数据卡片简洁，不做图表堆叠

手机端处理：

1. 统计卡片改为 2 列网格
2. 最近记录改成单列卡片
3. 缩略预览下沉到页面末尾

### 6.3 首页内容编辑页

这是整个后台最重要的页面。

桌面布局：

1. 左栏：模块树
2. 中栏：表单编辑区
3. 右栏：实时预览区

比例建议：

1. 左栏：`240-260px`
2. 中栏：主编辑区自适应
3. 右栏：`360-420px`

左栏模块树包含：

1. 导航栏
2. Hero
3. 技能栈
4. 项目
5. 联系方式
6. 页脚

中栏编辑区规则：

1. 每个模块按折叠卡片分组
2. 表单字段避免一屏过长
3. 使用清晰的小标题、说明文本和即时校验

右栏预览区功能：

1. 桌面 / 手机预览切换
2. 当前模块高亮预览
3. 保存后即时刷新

Hero 模块字段布局：

1. 左侧基础信息：称呼、姓名、描述
2. 右侧视觉信息：头像、按钮、打字机文案

技能模块字段布局：

1. 分类列表放左侧或上方
2. 当前分类内容放主编辑区
3. 支持批量录入标签

项目模块字段布局：

1. 项目列表在左侧
2. 当前项目详情在主区域
3. 技术栈采用 tag 输入样式

手机端处理：

1. 模块树改顶部 Tabs 或抽屉
2. 预览区改成单独切换页
3. 表单分组保持，但减少并列字段

### 6.4 技能栈管理页

目标：让技能分组和标签编辑足够顺手。

桌面布局：

1. 左侧分组列表
2. 中间分组编辑
3. 右侧效果预览

重点交互：

1. 拖拽分组排序
2. 拖拽技能排序
3. 批量粘贴技能
4. 等级拖动调整

视觉重点：

1. 技能标签使用半透明 pill 样式
2. 熟练度以小进度条或数字展示，不做夸张图表

手机端处理：

1. 分组列表顶部横向切换
2. 标签以单列块状展示

### 6.5 项目管理页

目标：把项目卡片做成最有展示感的编辑体验。

桌面布局：

1. 顶部筛选与排序
2. 中部列表 / 卡片双视图切换
3. 侧边抽屉或详情页编辑项目

列表视图适合：

1. 快速改排序
2. 快速开关显示状态
3. 查看链接完整性

卡片视图适合：

1. 看封面或图标效果
2. 看标题、技术栈和状态

项目编辑字段分组：

1. 基础信息
2. 展示形式
3. 技术栈
4. 外链
5. 状态与排序

视觉重点：

1. 项目卡片边距充足
2. 技术标签使用等宽字体
3. 状态标签小而清楚

手机端处理：

1. 默认只保留卡片视图
2. 编辑操作走全屏抽屉

### 6.6 联系方式管理页

目标：把联系卡片做成简单但精致的管理页。

布局：

1. 上方卡片列表
2. 下方或右侧编辑表单

字段：

1. 类型
2. 标签
3. 展示值
4. 链接
5. 图标类型
6. 图标资源
7. 排序

视觉重点：

1. 联系方式卡片在编辑端就能看出前台效果
2. 图标区有足够留白

### 6.7 媒体库

目标：作为本地上传资源的统一入口。

媒体库只管理本地资源，不展示外部 URL。

桌面布局：

1. 顶部工具栏
2. 中部网格列表
3. 右侧详情抽屉

顶部工具栏包含：

1. 上传按钮
2. 搜索
3. 视图切换
4. 排序方式

媒体卡片显示：

1. 缩略图
2. 文件名
3. 尺寸
4. 使用状态

详情抽屉显示：

1. 大图预览
2. 文件信息
3. 路径
4. 被哪些模块引用
5. 删除操作

视觉重点：

1. 网格卡片整齐，不拥挤
2. 选中态明显但不过度发光

手机端处理：

1. 单列卡片
2. 上传按钮固定在顶部
3. 详情使用底部抽屉

### 6.8 模板中心

模板中心是后台视觉重点页面。

目标：让用户有“挑主题”的感觉，而不是在改系统配置。

页面分层：

1. 模板列表页
2. 模板详情页
3. 模板配置页

#### 模板列表页

桌面布局：

1. 顶部筛选条
2. 中部大卡片网格
3. 当前使用模板固定高亮

模板卡片内容：

1. 封面图
2. 模板名
3. 风格标签
4. 支持模块
5. 当前状态

设计重点：

1. 卡片封面占比大
2. 卡片 hover 有轻微抬升和边框高亮
3. 当前启用模板有明显品牌标识

#### 模板详情页

布局：

1. 左侧大图预览
2. 右侧说明与配置摘要
3. 底部动作区

动作区按钮：

1. 预览模板
2. 使用模板
3. 返回列表

#### 模板配置页

布局：

1. 左侧配置分组
2. 中间配置表单
3. 右侧预览模拟器

配置分类：

1. 色彩
2. Hero
3. 卡片
4. 动效
5. 细节样式

手机端处理：

1. 模板列表改 1 列
2. 配置页改顶部分类切换
3. 预览单独切换展示

### 6.9 发布中心

目标：用最少步骤完成可控发布。

桌面布局：

1. 左侧版本差异区
2. 右侧发布检查区
3. 底部固定操作区

版本差异区显示：

1. 当前线上模板
2. 草稿模板
3. 最近变更摘要
4. 最近发布日期

检查区显示：

1. 模板兼容性
2. 资源完整性
3. 必填项检查
4. 发布备注输入

底部固定操作区：

1. 保存草稿
2. 预览
3. 发布
4. 回滚历史版本

视觉重点：

1. 不做复杂 diff 表格
2. 强调“状态明确”和“动作清楚”

手机端处理：

1. 改为上下结构
2. 底部固定主按钮

### 6.10 设置页

目标：统一管理站点基础配置和后台账号安全。

页面分组：

1. 站点信息
2. SEO 信息
3. 页脚与备案
4. 后台账号安全

账号安全包含：

1. 邮箱
2. 显示名称
3. 修改密码
4. 最后登录时间

## 7. 关键组件设计

### 7.1 主按钮

要求：

1. 主按钮使用品牌渐变
2. 圆角 `12px`
3. hover 时轻微上浮
4. 不使用过度辉光

### 7.2 次按钮

要求：

1. 透明底 + 细描边
2. hover 时背景略亮

### 7.3 表单输入框

要求：

1. 深色半透明背景
2. 清晰边框
3. 聚焦态使用青蓝色边框
4. 帮助文本紧跟字段下方

### 7.4 卡片

要求：

1. 卡片背景层级明显
2. 标题和描述分层清楚
3. 卡片内留白充足

### 7.5 状态标签

状态：

1. 草稿
2. 已发布
3. 未保存
4. 已启用
5. 已禁用

颜色要求：

1. 草稿：蓝灰
2. 已发布：绿色
3. 未保存：橙色
4. 危险操作：红色

## 8. 动效原则

后台动效必须服务于理解，不服务于炫耀。

保留动效：

1. 页面进入淡入
2. 面板切换滑入
3. 卡片 hover 微动
4. 模板卡 hover 高亮
5. 保存成功反馈

限制动效：

1. 不做大面积背景持续运动
2. 不做高频弹跳动画
3. 不做影响表单操作的悬浮效果

## 9. 桌面端与手机端视觉对照

### 9.1 桌面端

重点：

1. 空间感
2. 层级
3. 三栏效率
4. 模板预览的沉浸感

### 9.2 手机端

重点：

1. 信息顺序明确
2. 点击区域舒适
3. 不拥挤
4. 操作闭环完整

手机端不能做成：

1. 桌面三栏的缩略版
2. 字小按钮密的“工具页”
3. 预览和编辑混在一起的混乱界面

## 10. 交付要求

后台设计进入开发前，至少需要补齐以下内容：

1. 每个核心页面的桌面端线框
2. 每个核心页面的手机端线框
3. 颜色与字体 Token
4. 按钮、输入框、卡片、标签的基础样式
5. 模板中心卡片的高保真效果
6. 内容编辑页三栏布局稿

## 11. 最终结论

后台的“好看”不靠套一个现成管理模板，而靠以下几点共同成立：

1. 有统一设计语言
2. 有明确的信息层级
3. 有质感但克制的颜色和材质
4. 有清晰的桌面端与手机端适配策略
